## Phase 1e-A shipped — polymorphic packer stub ✅

Cumulative ~1300 LOC across 9 commits (Tasks 1-9 of the
`2026-05-07-phase-1e-a-implementation.md` plan).

New tree: `pe/packer/stubgen/` with sub-packages:
- `amd64/` — Builder wrapping `github.com/twitchyliquid64/golang-asm`
  (NEW dep, BSD-3); supported instructions: MOV / LEA / XOR / SUB /
  ADD / DEC / JMP / JNZ / CALL / RET / NOP. Cross-checked via
  `golang.org/x/arch/x86/x86asm.Decode`.
- `poly/` — SGN metamorphic engine: substitution table (XOR ↔
  SUB-neg ↔ ADD-complement), randomized RegPool over 14 GPRs,
  density-gated junk insertion, N-round (1..10) Engine.EncodePayload.
- `stage1/` — Round.Emit writes one decoder loop into the builder;
  multiple chained rounds use distinct `loop_N` labels.
- `host/` — minimal PE32+ emitter (DOS / PE / COFF / Optional /
  2-section table / file-aligned bodies). debug/pe parses cleanly.
- `stubvariants/` — maintainer Makefile + `stage2_main.go` source +
  initial committed `stage2_v01.exe` (~800 KB stripped).

Operator surface: `packer.PackBinary(payload, opts)` returns
`(host []byte, key []byte, err error)`. `cmd/packer pack -format=windows-exe`
wires the CLI; legacy `-format=blob` keeps the existing
encrypted-bytes flow.

Pre-return self-test (Go reference decoder) catches SGN encoding
bugs before the operator deploys. Per-pack output is byte-unique:
Hamming distance > 25% empirically on identical inputs with
different seeds.

Validated: all unit tests green; whole-module + cross-OS
(Windows/Darwin) builds clean. E2E execution test (run the
generated PE on a Windows VM, capture payload stdout) deferred to
the next session as it requires Windows VM scheduling.

## Don't lose

- `MALDEV_PACKER_RUN_E2E=1` is the production gate for actually
  jumping to OEP. Tests never set it; production builds opt in.
- notepad.exe FAILS to reflective-load (COMCTL32 SxS ordinal
  redirection). Use xcopy/where/find for E2E tests on Windows.
  Activation-context support is out of scope for Phase 1.
- `pe/packer/runtime` supports Windows PE (Phase 1b) and Go
  static-PIE ELF on Linux (Phase 1f). Non-Go ELFs (C/Rust/libc)
  return ErrNotImplemented — Stage E territory.
- `pe/packer.Options` has `Pipeline []PipelineStep` (Phase 1c).
  Zero-pipeline = AES-GCM backward-compat.

## Phase 1d shipped — anti-entropy ✅

`pe/packer/entropy.go` (~360 LOC) + `entropy_test.go` (16 tests).
New pipeline op `OpEntropyCover` (= 4) with three algorithms:

| Algo | Apparent entropy | Size | Use case |
|---|---|---|---|
| `EntropyCoverInterleave` | ~7.4 bits/byte (default 33% pad) | +50% | Real Shannon drop via low-entropy pad spliced between ciphertext chunks; pad pattern selectable (zeros / int3 / nop / mixed-asm). |
| `EntropyCoverCarrier` | unchanged bulk; first 32 bytes match PNG | +32B | Defeats "first-bytes look random" droppers heuristics. |
| `EntropyCoverHexAlphabet` | ≤4 bits/byte | ×2 | Each byte → 2 alphabet bytes (16-byte code-like alphabet: REX / mov / int3 / call / nop). Reversible via inverse LUT. |

**Operational stack to land below 5 bits/byte:** Interleave →
HexAlphabet, verified at ~4 bits/byte by `TestEntropyCover_StackedDeepDrop`.

The honest math correction shipped in `packer-design.md`:
the original "~4-5 bits/byte" claim for Interleave alone was
optimistic — Shannon caps a 33% mix at ~7.4. Stacking with
HexAlphabet is the operational answer.

## Phase 1f Stage E shipped — non-Go static-PIE end-to-end ✅

`pe/packer/internal/elfgate/elfgate.go` (broadened gate),
`pe/packer/runtime/runtime_linux.go` (alignDown for non-page-aligned
PT_LOAD), `pe/packer/runtime/testdata/hello_static_pie_c.s` (NEW —
hand-rolled asm fixture, ~20 lines), `Makefile` (NEW C target),
`main_test.go` (fixture-name env var dispatch), 4 reframed unit
tests + `TestPrepare_ELF_AcceptsRealNonGoStaticPIE` +
`TestRun_NonGoStaticPIE_E2E`.

Contract change:

- BEFORE Stage E: gate required `.go.buildinfo` section (Go-built
  binaries only).
- AFTER Stage E: gate is structural — `ET_DYN + no DT_NEEDED + ≥1
  PT_LOAD`. Hand-rolled asm, C/Rust built with `-static-pie`, and
  Go static-PIE all pass. `GoVersion` field populated for
  diagnostics when `.go.buildinfo` is present (purely informational).

Also relaxed: PT_LOAD vaddr need not be page-aligned. The mapper
now `alignDown`s segStart for `mprotect`. Real `gcc -static-pie -nostdlib`
output places the data segment at non-aligned vaddrs (e.g. 0x2f30);
the previous strict check was overconservative.

Field rename: `IsGoStaticPIE` → `IsStaticPIE` on
`elfgate.ELFHeaders` (unexported in `pe/packer/internal/elfgate`,
so no public API change).

Both fixtures load + run E2E:
- `hello_static_pie` (Go, prints "hello from packer" via Go runtime)
- `hello_static_pie_c` (asm, prints "hello from raw asm" via raw write+exit_group syscall)

Validated: linux unit tests + 2 E2E green; Windows + Darwin builds
clean.

## Phase 1f Stage C+D shipped — Go static-PIE end-to-end on Linux ✅

`pe/packer/runtime/elf.go` (+detection), `runtime.go` (+CheckELFLoadable),
`runtime_linux.go` (gate + Run() body), `runtime_linux_amd64.s` (NEW —
enterEntry asm), `testdata/hello_static_pie` (fixture). Plus
`pe/packer.ValidateELF` operator helper + cmd/packer pre-flight gate.
ELF gate logic extracted to `pe/packer/internal/elfgate` to break the
import cycle between `pe/packer` and `pe/packer/runtime`.

Z-scope contract:

- Accepts: ELF64 LE x86_64, ET_DYN, .go.buildinfo present, no
  DT_NEEDED. Built with
  `CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -buildmode=pie -ldflags='-s -w'`.
- Rejects everything else with ErrNotImplemented + rebuild hint.

Run() flow: 256 KiB MAP_PRIVATE|MAP_ANONYMOUS fake stack +
`/proc/self/auxv` passthrough with fresh AT_RANDOM canary +
AT_PHDR/AT_PHENT/AT_PHNUM patched to the loaded image's own phdrs
+ Plan 9 asm RSP swap + JMP to entry. Never returns on the happy
path (loaded binary calls exit_group, kills loader process).
Mirrors the Windows side's contract.

E2E test gated behind `go test -tags=maldev_packer_run_e2e`;
re-spawns the test binary with MALDEV_PACKER_E2E_INNER=1 so the
loaded fixture's exit_group doesn't kill the developer's go-test.

Validated: linux green (gated + ungated), Windows + Darwin builds
clean.

## Phase 1f Stage B shipped — Linux mmap + RELATIVE relocs ✅

`pe/packer/runtime/runtime_linux.go` (~225 LOC) replaces the Stage A
stub. ET_DYN PIEs now map cleanly:

- `unix.Mmap(MAP_ANONYMOUS|MAP_PRIVATE)` of total PT_LOAD span
  (page-aligned).
- File bytes copied per PT_LOAD; .bss tail zeroed by mmap.
- PT_DYNAMIC walked for DT_RELA / DT_RELASZ / DT_RELAENT;
  `R_X86_64_RELATIVE` applied as `*(u64*)(base+offset) = base + addend`.
- Per-segment `unix.Mprotect` to declared PF_R/W/X flags.
- `Free()` munmaps; idempotent on zero-base.

Rejected (Stage C/D territory): ET_EXEC (only PIE supported),
PT_INTERP (needs ld.so), PT_TLS (needs TLS init), any non-RELATIVE
reloc type (needs symbol resolution).

3 new sub-tests cover: ET_EXEC rejection, PT_INTERP rejection,
PT_TLS rejection, missing-PT_DYNAMIC rejection, happy-path map of
a synthetic minimal PIE (ET_DYN + one PT_LOAD covering the file +
PT_DYNAMIC with DT_NULL terminator), Free idempotency.

## Phase 1f Stage A shipped — ELF parser + format dispatch ✅

`pe/packer/runtime/elf.go` (~155 LOC) + Linux/long-tail platform
files split. [Prepare] now dispatches by magic byte:

- `MZ` → existing PE backend (Phase 1b on Windows;
  [ErrFormatPlatformMismatch] on Linux).
- `\x7fELF` → new ELF backend (Linux returns [ErrNotImplemented]
  with parsed-but-not-mapped [PreparedImage] carrying entry +
  size; Windows returns [ErrFormatPlatformMismatch]; long-tail
  OSes return [ErrNotWindows]).

Sentinels added: `ErrBadELF`, `ErrUnsupportedELFArch`,
`ErrNotELFExec`, `ErrFormatPlatformMismatch`, `ErrNotImplemented`.
Existing `ErrNotWindows` promoted from `runtime_stub.go` to
cross-platform `elf.go`.

8 new sub-tests cover: too-short input, unknown magic, ELF32
class, big-endian data, non-x86_64 machine, ET_REL type, missing
PT_LOAD, truncated phdr table, GOOS-conditional backend surface
(linux → ErrNotImplemented + parsed image; windows → mismatch;
other → ErrNotWindows), PE-on-Linux mismatch.

Stages remaining: B (mmap + RELA), C (ld.so resolution via
dlopen/dlsym), D (TLS init + jump-to-entry).
