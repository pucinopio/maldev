# Tutorial 01 — Issue a licence in the TUI, verify it in your binary

> The shortest end-to-end path from "I have license-manager
> open" to "my binary refuses to start unless the operator hands
> it a valid PEM and the matching issuer public key".
>
> **What the operator does:** open the TUI, generate an issuer,
> issue a licence, export the PEM + the issuer's public key.
>
> **What the developer does:** drop `client/main.go` into their
> binary, ship the public key alongside, refuse to start without
> both files.

## Pieces

| File | Role |
|---|---|
| TUI session | Where the operator generates the licence + public key. |
| [Client program](https://github.com/oioio-space/maldev/tree/master/examples/license-manager/tutorials/01-issue-and-verify/client) | The Go program a developer pastes into their binary. |

## TUI walkthrough (the operator side)

Boot the manager once: `go run ./cmd/license-manager`. Pass a
passphrase via env (`MALDEV_MGR_PASSPHRASE=demo`) or type it at
the modal. The DB lives in `~/.config/license-manager/db.sqlite`.

The first launch wizard creates the first issuer; subsequent
launches land on the dashboard.

1. **Switch to the Licences screen** — key `2`.
2. **Open the new-licence wizard** — key `n`.
3. **Step 1 (Identity)** — the active issuer is pre-selected
   (`>>` ASCII marker). Hit Enter.
4. **Step 2 (Recipient)** — leave at "none" if you don't need a
   sealed payload. Enter.
5. **Step 3-4 (Machine / Binary)** — skip with Ctrl+→ if you
   don't want to bind to those (this tutorial issues an
   unbound licence).
6. **Step 5 (Validity)** — defaults: now + 1 year. Enter.
7. **Step 6 (FreeFields)** — set `subject = alice@example.com`,
   `audience = demo`. Enter.
8. **Step 7 (TOTP)** — leave OFF. Enter.
9. **Step 8 (Review)** — confirm with Enter. The licence is
   signed and persisted; the wizard closes and the new row
   shows up in the table with the `>>` cursor on it.

Now export both halves:

10. On the new row, key `E` → type a path (`/tmp/alice.license`)
    → Enter. The PEM lands on disk.
11. Tab `3` (Issuer keys) → `E` on the active issuer →
    `/tmp/issuer.pub` → Enter. The public key PEM lands on disk.

You now have everything a deployed binary needs.

## Developer integration (the client side)

The client program is a 60-line Go binary. The whole interface
the licensed binary needs is two files and one function call:

```go
licPEM, _ := os.ReadFile("license.pem")
pubPEM, _ := os.ReadFile("issuer.pub")

pub, kid, _ := licensekg.ParsePublicKey(pubPEM)
trusted := licensekg.Trusted{Keys: licensekg.SingleKey(kid, pub)}

v, err := licensekg.Verify(licPEM, trusted)
if err != nil {
    log.Fatalf("license verification failed: %v", err)
}
log.Printf("running for %s (expires %s)", v.Subject, v.NotAfter)
```

Bound licences add evidence options
(`license.WithMachineID(...)`, `license.WithPassword(...)`,
`license.WithTOTPCode(...)` — see
[Tutorial 02](./02-bindings-and-verify.md) *(coming)*).

The reference binary builds and runs standalone:

```bash
# From the repo root.
go build -o /tmp/license-check ./examples/license-manager/tutorials/01-issue-and-verify/client

/tmp/license-check --license /tmp/alice.license --issuer-pub /tmp/issuer.pub
# [ok] licence verified
#      subject:  alice@example.com
#      issuer:   lab (key-id tutorial-01)
#      features: [basic]
#      audience: [demo]
#      not_after: 2027-05-28
```

A tampered PEM exits non-zero with `[fail] verify: signature
mismatch`. The licensed binary should treat that as a
fatal-at-startup error.

## How to keep the tutorial honest

The matching `e2e_test.go` renders the VHS tape via
`cmd/tui-gif` AND runs the client against a real licence the
test issues through `service.Services` — the same code path the
wizard runs. If either half breaks, the test fails and the
operator-facing docs cannot ship.

```bash
go test ./examples/license-manager/tutorials/01-issue-and-verify
```
